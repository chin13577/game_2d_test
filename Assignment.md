# Assignment

## Introduction

This assignment serves to evaluate your programming, problem-solving, software design, and creativity in game development. Your task is to provide a solution demonstrating your understanding of programming principles, clean and efficient code, and best practices. The assignment will be in the form of a simple GDD. Please read this document thoroughly and carefully before proceeding.

## Technical requirements

This section describes requirements on a technical level. The following requirements are to be met:

- Using the latest Unity LTS version.
- Using only legally usable assets.
  - [Assets · Kenney](https://kenney.nl/assets) is recommended but not required.
- Your code must be entirely your own work, without any AI assistance.
  - This includes chat bots (ChatGPT, Google Gemini, etc.) and code-assist AI (Copilot, Tabnine, etc.).
- The build must be able to build as a Windows executable.
- Unity's packages are allowed. Third-party plugins are not allowed. If in doubt, don't use it.
- Any technical decision must be logged in a text file that describes what needs to be decided, what the thought process is, and what the conclusion is.
- Extra features are welcome.
- Trust your judgment if any requirements seem unclear. Just make sure the final result is fun to play.

## Evaluation

This section describes a general idea of how we evaluate candidate assignments.

- This assignment should be finished within the time limit.
- All required requirements should be met.
- The score will be based on many factors, including but not limited to:
  - Functionality: The project must function as intended.
  - Cleanliness: The project and code must be written cleanly.
  - Reusability: Code must be able to be re-used.
  - Efficiency: The code must be efficient.
  - Customizable: The project must be easy to customize and make changes to.
  - Best practices: Code should always follow industry best practices.
  - Documentation: Important decisions must be logged.
  - User experience: Your software must be easy to understand.
- The utilization of aesthetically pleasing assets will not influence your evaluation score.
- You can earn extra points by making your assignment resemble a real game more closely. For example, consider adding audio, animations, or particle effects.
  - However, extra points are given less priority compared to the core scoring methods.
- Documents are preferably in English. Only English and Thai are acceptable.

## Submission

This section describes how to submit your assignment.

- The project must be stored in version control, preferably Git.
- The project must be able to open using Unity's latest LTS without any problem.
- Alongside the project, you must include the technical decision document in version control.
  - Technical decisions can be in a raw text file or Markdown.
